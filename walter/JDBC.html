JDBC Reference

Purpose:
The JDBC Endpoint allows your Mule application to use JDBC to communicate with a database.

Discussion:
A JDBC Endpoint lets an application read from and write to a database that supports the JDBC driver protocol. Use the JDBC Endpoint to perform create, read, update, and delete (often referred to by the acronym CRUD) operations on database records. You can also use this endpoint to invoke database stored procedures, such as procedures to create new tables dynamically.

When used as an outbound endpoint, you can think of a JDBC Endpoint as essentially a processor to execute SQL code. The expected return data from executing SQL code is a list of maps.

The JDBC inbound endpoint is better used for making changes to the database and for invoking recurring processing tasks that occur when certain database events are triggered.
