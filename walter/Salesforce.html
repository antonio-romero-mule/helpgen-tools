Salesforce Reference

Purpose:
The Salesforce Cloud Connector allows your Mule application to access the Salesforce CRM application APIs.

Discussion:
To access the Salesforce APIs, you must create a developer account with salesforce.com. Sign up for an account at http://developer.force.com/ to create an account and get a user name, password, and security token.

You also must have created a Mule Studio global element for the Salesforce cloud connector. You can do so from the Global Element pane prior to dropping the Salesforce Cloud Connector on the canvas. Or, you can create the global element from within a Salesforce Pattern Properties dialog, when setting the config reference property.

Set the Config Reference and Operation properties from the General tab in the Salesforce Pattern Properties dialog:

* Select the Salesforce global element for the configuration reference from the Config Reference pull-down list. If you have not yet created a Salesforce global element, click the Add button to create this element. (See Salesforce Global Element.) * Select a Salesforce operation from the Operation pull-down list. Operations correspond to the available Salesforce APIs.

After you select an operation, the bottom half of the screen displays the properties that you set for the selected operation. (See the figure below.)

Properties marked with a red x are required and others are optional. For example, if you select Get Deleted Range for the operation, the Pattern Properties pane displays the properties for this operation, including:

  • Type
  • Start Time
  • Stop Time

Mule Studio provides a default value, order, for the Type property. Whenever possible, Mule Studio provides default property values. You can change any such default value. Notice that both Start Time and Stop Time, the delimiters for the date range, are required. Times are entered using UTC format, such as: 12:45:15, for hour, minute, and second. Note that the API uses only the hour and minute fields for the time.

When you select the Create operation, you use the Advanced tab to add and edit sObjects. The figure below illustrates these steps.

Click the plus icon to add a new salesforce sObject. Select that object, then click the edit icon to add key/value pairs and value references for the object. Click the red x icon to delete a sObject or to delete a key/value pair.

Salesforce Global Element

Click the Add button next to the Config reference field in the Pattern Properties dialog to create a Salesforce global element. (You can also do this from the Global Elements pane.) If you have selected a Salesforce global element from the Config reference pull-down list, you can click the Edit button to modify configuration values in that global element.

You enter the Salesforce authentication information for the global element properties.

  • Name: The name of the global element. Mule Studio provides a default name which you can change.
  • Username: The user name for your salesforce.com account.
  • Password: The password for your salesforce.com account.
  • Security token: The security token provided to you by salesforce.com.
