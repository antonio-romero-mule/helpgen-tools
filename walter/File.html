File Reference

Purpose:
The File Endpoint enables reading and writing message data to and from a file system.

Discussion:
When File Endpoint is used to receive data (as an inbound endpoint), you can read the data in files from the file system. You can also use File Endpoint to move backup files to a specific directory or folder.

When you read data from a file located on the file system through an Inbound File Endpoint, that file is deleted after it is read. You can keep a copy of the file after it is read by setting the Move to directory property to a location where you want the copy placed. You can also specify the Move to pattern property to change the file name when it is moved.

  • Reading Large Files: If you anticipate reading a file containing a large amount of data, you may want to set an amount of time that Mule ESB should wait before reading and processing the file. Having Mule wait before processing ensures that all data is written to the file. Use the File age property to set this wait time (in milliseconds).
  • Sorting Files: You can also control the sort order in which the File Endpoint reads files. To do so, set the Comparator property to a Java comparator class that specifies how to sort incoming files. You can use any of the comparator classes provided by Mule, for example: org.mule.transport.file.comparator.OlderFirstComparator. If you write your own Java comparator class, be sure that the class implements the java.util.Comparator interface. When sorting incoming files, you can also reverse the sort order by checking the Reverse order property.

For an Outbound File Endpoint you set just two path-related properties: The path to the output file folder or directory location and a pattern to apply to the output file name. If you don't specify an output pattern, the output file has the same name as the input file.
